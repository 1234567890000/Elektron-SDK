MarketMaker Update Message
A MarketMaker update message is encoded using UpdateMsg and sent by OMM interactive provider and OMM non-interactive provider applications. Updates will not be received before images, and a true snapshot is supported.
The provider can send an update message to add, update, or remove market maker information.
Note:
The provider should only send the Name and ServiceName in the first Refresh response message. However if EMA is configured MsgKeyInUpdates with true value, then the Name and ServiceName must be provided for every Update response messages.

Name

Optional (Required if MsgKeyInUpdates is set to true). This should be the ID of the service that provides the data.

The provider application should set either the ServiceName or ServiceId of the service, but not both.

Optional (Required if MsgKeyInUpdates is set to true). This should be the ID of the service that provides the data.
Note:
The provider application should set either the ServiceName or ServiceId of the service, but not both.
Payload
Required.
A MarketMaker is represented by a Map, where each entry (MapEntry) contains FieldList which has information about a market maker.
