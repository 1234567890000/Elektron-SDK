Reactor Error Handling
The RsslErrorInfo structure is used to return error or warning information to the application. This can be returned from the various Reactor functions as well as part of a callback function. When returned directly from a Reactor function, this indicates that an error occurred while processing in that function. If returned as part of a callback function, this indicates that an error has occurred on one of the channels that the Reactor is managing.
The RsslErrorInfo members are described in the following table.
 
Table 3: RsslErrorInfo Structure Members  
STRUCTURE MEMBER
DESCRIPTION
rsslErrorInfoCode
An informational code about this error. Indicates whether it reports a failure condition or is intended to provide non-failure related information to the user. For details on available codes, refer to Table 4.
rsslError
The underlying error information from the Transport API. This includes a pointer to the RsslChannel on which the error occurred, both a Transport API and a system error number, and more descriptive error text. The RsslError and its values are described in the Transport API C Edition Developers Guide.
errorLocation
Provides information about the file and line that the error occurred at. Detailed error text is provided via the rsslError portion of this structure.
RsslErrorInfo.errorLocation length is limited to 1,024 bytes.
