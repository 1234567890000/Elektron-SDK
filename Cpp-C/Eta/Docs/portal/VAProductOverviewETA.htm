Overview

Return to Top

The Value Added Components provide alternative entry points for applications to leverage the OMM-based Transport API and are intended to simplify and compliment the use of the Transport API. The Value Added Components allow the application developer to choose and customize the balance between ultra high performance raw access and ease of use feature functionality. The choice to leverage these components is up to the application developer; Value Added Component use is not required to use the Transport API. In addition, the application can utilize some or all of the Value Added Components. Value Added Components are written to the Transport API interfaces and are designed to work alongside the Transport API. As a result, the interfaces have a similar look and feel to Transport API interfaces in order to provide simple migration and consistent use between all Value Added Components and the Transport API.

Product Features

Return to Top

The Transport API Valued Added Components contain the following components:

  • Transport API Reactor Component
    • This is a callback-based component that provides connection management and event processing features for OMM-based applications.
    • Supports OMM-based Consumers, Providers and Non-Interactive Providers.
    • Provides domain-specific callbacks for the Login, Source Directory and Dictionary RDM representations.
  • Administrative Domain Model Representations
    • This component provides RDM-specific representations of the OMM Administrative domain models.
    • Provides structures that represent specific messsages defined in the Transport API RDM Usage Guide document for the Login, Source Directory and Dictionary domains.
    • In addition, this component handles encoding and decoding of the OMM Administrative domain messages, which significantly reduces the amount of code necessary to interact with those domain models.
  • Value Added Utilities
    • This component is a collection of helper constructs, mainly used by the Transport API Reactor.
  • Value Added Payload Cache
    • This component provides a cache for storing and retrieving encoded OMM payload data. Each cache container is a collection of payload entries, which can store the following OMM payload types: Field List, Element List, Map, Vector, Series, Filter List.
