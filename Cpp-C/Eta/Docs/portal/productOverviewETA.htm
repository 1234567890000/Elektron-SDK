Transport API C Edition - Documentation Portal

Overview

Return to Top

The Transport API is a low-level API that provides the most flexible development environment to the application developer. It is the foundation on which all Thomson Reuters Open Message Model(OMM) -based components are built. By utilizing an API at the Transport level, a client can write to the same API as the ADS / ADH and achieve the same levels of performance provided by the Core Infrastructure. The Transport API is thread safe and thread aware. It is capable of handling all OMM primitives and all Domain Models - this includes Thomson Reuters defined models as well as user defined models.

Product Features

Return to Top

The Transport API provides the following functionality to all applications:

  • Ability to fragment and reassemble large messages.
  • Ability to pack multiple small messages into the same network buffer.
  • Ability to choose a thread locking model based on application needs. This allows clients to develop single-threaded, multi-threaded thread safe, and thread-aware solutions.
  • Full control of the number of available message buffers, as well as the ability to change that number during runtime.
  • No configuration, message file, or log file dependencies. Everything is programmatic.
  • Allows users to send messages with different priority levels.
  • The following transport configurations are supported:
    • TCP socket
    • Reliable UDP Multicast
    • HTTP
    • HTTPS
    • Unidirectional Shared Memory

Additionally, the Transport API provides the following application-specific capabilities:

  • Consumer Applications:
    • Ability to make Streaming and Snapshot based subscription requests.
    • Ability to make OMM Batch, View, and Symbol List subscription requests to capable providers.
    • Support for Pause and Resume functionality on active data streams to capable providers.
    • Ability to send Post Messages used for Consumer-based Publishing and contributions to capable providers.
    • Ability to send and receive Generic Messages.
    • Ability to consume data with a tunneled HTTP connection to an ADS via WinINet(Windows platforms only).
  • Interactive Provider Applications:
    • Ability to receive requests and respond to Streaming and Snapshot based subscription requests (Previously known as Managed or Sink-Driven Server applications).
    • Ability to receive and respond to requests for Batch, View, and Symbol List subscription requests.
    • Ability to receive Pause and Resume requests on active data streams.
    • Ability to receive acknowledge Post Messages used for Consumer-based Publishing and contributions.
    • Ability to send and receive Generic Messages.
    • Ability to write a server-based application that accepts multiple connections, or allows multiple consumers to connect to a provider.
  • Non-Interactive Provider Applications:
    • Ability to start up and publish data directly to ADH (Previously known as Source-Driven or Broadcast style server applications).

This release also includes the Transport API Value Added components. These components are intended to simplify the Transport API usage, and reduce the amount of code required to create OMM-based applications. In addition to the previous Transport API features, these components provide the following additional features:

  • The Transport API Reactor component is a connection management and event processing component.
  • The Administration Domain Model Representations reduce the code required to interact with the RDM administrative domains.
  • The Value Added Utilities are a collection of helper constructs mainly used by the Transport API Reactor.
Transport API Documentation

Return to Top

The following documents are included in this release.

Transport API Developers Guide

This document provides information and examples intended to aid in development with the Transport API product. It provides detailed descriptions for OMM Data and Message package concepts and use of the RSSL Reliable Transport.

Transport API-RDM Usage Guide

This document describes the message layout and use for Thomson Reuters provided domain message models. This content is available through the Thomson Reuters Enterprise Platform, EleKtron, Reuters Data Feed Direct, and various other sources.

Transport API Reference Manual

This is a doxygen generated reference manual, based on the comments and usage information provided in the Transport API header files. This is intended As a supplement to the Transport API Developers Guide.

Transport API Performance Tools Guide

This document details the Transport API Open Source Performance Tool Suite, their usage, and their design and behaviour.

Transport API Value Added Components Portal

This is the main documentation portal for the Transport API Value Added Components.

Utilities Documentation

Return to Top

The following documents are included in this release.

DACS Lock Reference Manual

This document describes DACS Lock creation and manipulation of permission information. The DACS Lock package is made available as a supplement to the Transport API so applications can properly create any necessary permission data for content they provide.

ANSI Reference Manual

This document describes ANSI Page encoding and decoding. The ANSI package is made available as a supplement to the Transport API so, if needed, an application can provide or consume ANSI Page based content.
